# 9 种 Skill 类型详解
> 每种类型的适用场景、设计模式、模板和评估重点

---

## 类型 1：库和 API 参考

**适用场景：** 内部库、CLI、SDK、框架的使用指南
**设计模式：** Tool Wrapper
**模板：** `templates/library-reference.md`

### 核心特征
- 教 Claude 正确使用特定工具/API
- 包含安装、配置、常用命令
- 踩坑清单（Gotchas）是核心章节
- 好坏对比示例

### 必须包含
- 快速入门（安装 + 第一个示例）
- 常用 API/命令 表格
- 踩坑清单（至少 3 个陷阱）
- 边界情况处理

### 评估重点
- API 覆盖完整性
- 示例代码正确性
- 陷阱识别准确度

---

## 类型 2：产品验证

**适用场景：** 测试流程、断言验证、发布前检查
**设计模式：** Pipeline + Reviewer
**模板：** `templates/product-verification.md`

### 核心特征
- 多步骤顺序执行
- 硬性门控（Gating Instructions）
- 进度追踪（progress.yaml）
- 自动化 + 手动验证结合

### 必须包含
- 验证目标表格
- Pipeline 步骤流程（至少 4 步）
- 通过标准
- 问题报告格式

### 评估重点
- 步骤完整性
- 门控有效性
- 报告格式规范性

---

## 类型 3：数据获取与分析

**适用场景：** 数据查询、报表生成、趋势分析
**设计模式：** Generator
**模板：** `templates/data-analysis.md`

### 核心特征
- 数据源定义清晰
- 查询模板标准化
- 报表格式统一
- 数据准确性检查

### 必须包含
- 数据源概述
- 常用查询（含 SQL/查询语句）
- 报表生成模板
- 数据可视化建议

### 评估重点
- 查询准确性
- 报表完整性
- 数据口径一致性

---

## 类型 4：代码生成

**适用场景：** 规格转代码、设计稿转前端、Schema 转模型
**设计模式：** Generator + Pipeline
**模板：** `templates/code-generation.md`

### 核心特征
- 输入规格解析
- 模板填充机制
- 代码规范自动应用
- 质量检查（lint）

### 必须包含
- 输入要求（格式 + 必填字段）
- Pipeline 步骤（解析→规划→生成→验证→输出）
- 代码规范章节
- 生成示例

### 评估重点
- 生成代码正确性
- 规范遵循度
- 模板覆盖度

---

## 类型 5：文档生成

**适用场景：** PR 描述、API 文档、发布说明、技术文档
**设计模式：** Generator + Inversion
**模板：** `templates/doc-generation.md`

### 核心特征
- 标准化文档结构
- 输入→输出的生成规则
- 语言风格统一
- 字数/格式限制

### 必须包含
- 输入要求（必需 + 可选信息）
- 文档结构模板
- 生成规则
- 语言风格指南

### 评估重点
- 文档结构完整性
- 语言简洁性
- 格式规范性

---

## 类型 6：代码审查

**适用场景：** 安全检查、性能分析、代码质量
**设计模式：** Reviewer + Tool Wrapper
**模板：** `templates/code-review.md`

### 核心特征
- 多维度检查清单
- 严重级别分类
- 修复建议具体
- 工具集成（Semgrep/ESLint）

### 必须包含
- 检查清单（安全/性能/质量）
- 审查流程（3 步）
- 报告格式（高危/中危/建议）
- 踩坑清单

### 评估重点
- 问题发现率
- 修复建议质量
- 误报率

---

## 类型 7：部署流程

**适用场景：** 部署、发布、回滚
**设计模式：** Pipeline + Reviewer
**模板：** `templates/deployment.md`

### 核心特征
- 严格顺序执行
- 部署前检查
- 回滚机制
- 监控观察

### 必须包含
- 环境列表
- Pipeline 步骤（至少 5 步）
- 回滚流程
- 部署检查清单

### 评估重点
- 流程完整性
- 回滚可用性
- 监控指标覆盖

---

## 类型 8：团队规范

**适用场景：** 编码风格、提交规范、命名规范
**设计模式：** Tool Wrapper
**模板：** `templates/team-norms.md`

### 核心特征
- 规范表格化
- 正反对比示例
- Git 提交规范
- 术语统一

### 必须包含
- 命名规范（变量/函数/文件）
- 代码格式（缩进/行长度）
- 注释规范
- Git 提交规范

### 评估重点
- 规范完整性
- 示例清晰度
- 可执行性

---

## 类型 9：调研整理

**适用场景：** 技术调研、竞品分析、方案选型
**设计模式：** Inversion + Generator
**模板：** `templates/research.md`

### 核心特征
- 信息源优先级
- 观点对比分析
- 来源标注
- 推荐方案

### 必须包含
- 调研范围（核心 + 边界）
- 信息来源（优先级排序）
- 调研流程（4 步）
- 报告结构

### 评估重点
- 信息覆盖度
- 分析深度
- 建议合理性

---

## 类型选择决策树

```
用户请求创建 Skill
│
├─ 关于工具/库/API 的使用？ ──────────→ 类型1: 库和API参考
├─ 关于测试/验证产品功能？ ──────────→ 类型2: 产品验证
├─ 关于查询数据/生成报表？ ──────────→ 类型3: 数据获取与分析
├─ 关于根据规格生成代码？ ──────────→ 类型4: 代码生成
├─ 关于生成标准化文档？ ──────────→ 类型5: 文档生成
├─ 关于审查代码质量/安全？ ──────────→ 类型6: 代码审查
├─ 关于部署/发布/回滚？ ──────────→ 类型7: 部署流程
├─ 关于编码规范/提交规范？ ──────────→ 类型8: 团队规范
├─ 关于技术调研/知识整理？ ──────────→ 类型9: 调研整理
│
└─ 不确定 → 问用户 5 个关键问题（见 workflow.md 阶段1）
```
