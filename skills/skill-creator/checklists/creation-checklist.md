# Skill 创建检查清单
> 创建新 Skill 时，逐项检查确保质量

## 阶段 1-2：构思与验证

- [ ] 确认任务复杂且经常重复（≥ 5 次）
- [ ] 确认已有工具可以完成任务
- [ ] 确认不是 Agent 天生就会的简单任务
- [ ] 确认 Skill 能带来明显价值

## 阶段 3：用例定义

- [ ] 定义了 2-3 个具体使用场景
- [ ] 定义了触发条件（trigger words）
- [ ] 定义了不适用的场景
- [ ] 定义了输入输出格式

## 阶段 4：模式选择

- [ ] 选择了合适的设计模式（9 选 1 或组合）
- [ ] 确认模式与 Skill 类型匹配
- [ ] 如使用 Pipeline，定义了步骤和检查点

## 阶段 5：生成草稿

- [ ] SKILL.md 包含 YAML frontmatter
- [ ] description 足够精准（能正确触发）
- [ ] 使用了 DO NOT / NEVER 防止错误行为
- [ ] 包含具体示例（输入→输出）
- [ ] 包含踩坑清单（Gotchas）
- [ ] 引用了 assets/ 中的模板

## 阶段 6：文件组织

- [ ] 主文件 < 500 行
- [ ] 大型资源拆分到 references/
- [ ] 模板存放在 templates/ 或 assets/
- [ ] 脚本存放在 scripts/
- [ ] 目录结构清晰

## 阶段 7：内部审查

- [ ] 使用了 skill-creator 自身进行审查
- [ ] 检查了所有链接和引用
- [ ] 验证了所有代码示例
- [ ] 测试了触发词准确性

## 阶段 8：测试执行

- [ ] 编写了 evals.json（至少 3 个用例）
- [ ] 每个用例有 2-4 条期望
- [ ] 期望具体可验证
- [ ] 执行了所有测试用例
- [ ] 记录了 metrics 和 timing

## 阶段 9：评估与迭代

- [ ] Grader 评分通过率 ≥ 80%
- [ ] 修复了所有高危问题
- [ ] 更新了版本历史
- [ ] 用户确认满意

## 阶段 10：归档

- [ ] 更新了 Skill 版本号
- [ ] 更新了最后修改日期
- [ ] 清理了临时文件
- [ ] 文档完整
