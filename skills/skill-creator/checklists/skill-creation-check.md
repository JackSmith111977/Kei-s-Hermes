# Skill 创建过程检查
> 在创建 Skill 的过程中，随时检查是否遵循最佳实践

## 触发条件检查

- [ ] **description 精准** — 能正确触发，不会误触发
- [ ] **触发词明确** — 用户知道如何触发
- [ ] **边界清晰** — 知道什么情况下不用

## 内容质量检查

- [ ] **指令具体** — 不说"输出高质量结果"，说"输出包含 X、Y、Z"
- [ ] **DO NOT 清单** — 列出了绝对不要做的事
- [ ] **示例完整** — 每个关键点都有示例
- [ ] **示例可运行** — 代码示例可以直接复制运行

## 结构检查

- [ ] **主文件精简** — SKILL.md < 500 行
- [ ] **资源分离** — 大型模板/参考文件已拆分
- [ ] **链接有效** — 所有 \`{{asset:xxx}}\` 引用存在
- [ ] **目录清晰** — references/templates/scripts 分类明确

## 设计模式检查

- [ ] **模式匹配** — 选择的模式适合 Skill 类型
- [ ] **Pipeline 门控** — 如使用 Pipeline，有硬性门控
- [ ] **进度追踪** — 如使用 Pipeline，有 progress.yaml
- [ ] **检查点定义** — 每步有明确的通过标准

## 评估检查

- [ ] **evals.json 存在** — 定义了测试用例
- [ ] **期望可验证** — 不是"质量好"而是"包含 X"
- [ ] **覆盖全面** — 正常 + 边界 + 错误
- [ ] **Grader 评分** — 有评分结果

## 安全性检查

- [ ] **无密钥泄露** — 不含密码、Token、API Key
- [ ] **无敏感路径** — 不含绝对路径
- [ ] **输入验证** — 处理了异常输入
- [ ] **沙箱兼容** — 脚本在沙箱中可运行
