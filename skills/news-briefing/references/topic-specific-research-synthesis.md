### 3. 交叉验证操作

**示例：台湾问题在会谈中的处理**

| 来源 | 表述 | 可信度 |
|:----|:-----|:------:|
| 新华社 | 习近平强调台湾问题是"最重要的问题"，处理不当将"导致冲突" | 🥇 官方纪要 |
| ABC News (US) | 白宫会谈纪要"未提及台湾"，特朗普被问及未回应 | 🥇 随团记者 |
| CNN | Xi warned of "conflict" if Taiwan handled improperly | 🥇 现场报道 |
| 美方官方纪要 | 无台湾相关内容 | 🥇 官方文件 |

**结论**：中方高度重视台湾议题并公开表态，美方有意识地避免在官方纪要中记录——这是重要的叙述差异，必须在报道中同时呈现双方版本。

### 4. 主题组织结构（实战）

将采集的 15+ 来源内容组织为以下主题板块：

```
特朗普访华专题报道
├── 头条：习近平与特朗普历史性会晤（核心事实）
├── 数据条：5 个关键数字（贸易额/关税/时间跨度）
├── 核心议题（4 篇双栏）
│   ├── 贸易经济：波音订单、农产品、能源、投资委员会
│   ├── 伊朗局势：霍尔木兹海峡、中国承诺
│   ├── 台湾问题：习近平警告 vs 美方纪要沉默
│   └── 科技与AI：H200芯片、AI安全框架、黄仁勳随行
├── 编辑点评（总结性点评）
├── 中美关系新定位："建设性战略稳定关系"
├── 访问详细行程（时间线）
├── 随访商界阵容
├── 历史脉络时间线（2025.1 - 2026.12）
├── 各方反应（美国国内/中国/国际/伊朗）
└── 未来展望（短期/中长期路线图）
```

## PDF 排版本战教训

### 系统 Python vs venv

```bash
# ✅ WeasyPrint 安装在系统 Python 中
$ /usr/bin/python3 -c "from weasyprint import HTML; print('OK')"
OK

# ❌ venv 的 python3 中找不到
$ python3 -c "from weasyprint import HTML; print('OK')"
ModuleNotFoundError: No module named 'weasyprint'

# 正确执行方式
$ /usr/bin/python3 generate_news_pdf.py
```

### 报纸 CSS 排版设计要点

| 关注点 | 实战配置 |
|:------|:---------|
| 报头 | `font-size: 38pt; letter-spacing: 12px;` + `border-bottom: 3px double` |
| 头条 | `24pt` 粗体 + `12pt` 副标题 + 8pt 元信息 |
| 字体层级 | 报头 > 头条 > 栏目标题(13pt) > 正文(8.5-10pt) > 来源(7-8pt) |
| 双栏 | CSS `column-count: 2`（勿用 flexbox，分页会断） |
| 数据条 | 深蓝背景 5 栏等宽，白色文字 |
| 表格 | 深蓝表头 + 白色文字 + 交替行背景 |
| 编辑点评 | 淡黄背景 `#fff8e1` + 金色左边框 `#d4a017` |
| 三栏快讯 | `column-count: 3`，用于页底短新闻集锦 |
