# 学习质量评分细则

## 评分公式

```
质量分 = 信息覆盖度(30) + 交叉验证(25) + 可操作性(25) + 结构完整度(20)
```

满分 100 分。

---

## 维度 1：信息覆盖度（0-30 分）

| 分数 | 标准 |
|------|------|
| 25-30 | 覆盖了主题的各个方面，包含核心概念、最新进展、实践案例 |
| 18-24 | 覆盖了主题的主要方面，缺少 1-2 个重要维度 |
| 10-17 | 只覆盖了主题的部分方面，缺少多个重要维度 |
| 0-9 | 信息严重不足，只找到 1-2 个来源 |

**检查项**：
- [ ] 搜索了 ≥3 个高质量来源
- [ ] 覆盖了官方文档/权威社区
- [ ] 包含了时效性信息（版本、日期）

---

## 维度 2：交叉验证得分（0-25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 核心结论有 ≥3 个独立来源佐证 |
| 15-19 | 核心结论有 2 个独立来源佐证 |
| 8-14 | 核心结论只有 1 个来源，标注"待验证" |
| 0-7 | 无交叉验证，直接使用单一来源 |

**检查项**：
- [ ] 每个核心结论至少 2 个来源确认
- [ ] 矛盾信息以官方文档为准
- [ ] 标注了不确定信息

---

## 维度 3：可操作性（0-25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 提炼了明确的步骤、配置示例、避坑指南 |
| 15-19 | 有部分操作步骤，但缺少示例或避坑 |
| 8-14 | 只有理论描述，缺少可操作的步骤 |
| 0-7 | 纯信息堆砌，无可操作内容 |

**检查项**：
- [ ] 有具体的操作步骤或配置示例
- [ ] 有避坑指南/常见错误
- [ ] 有应用场景说明

---

## 维度 4：结构完整度（0-20 分）

| 分数 | 标准 |
|------|------|
| 16-20 | 结构清晰：核心概念→特性→最佳实践→对比→学习路径 |
| 12-15 | 结构基本完整，缺少 1 个部分 |
| 6-11 | 结构不完整，信息组织混乱 |
| 0-5 | 无结构，纯文本堆砌 |

**检查项**：
- [ ] 使用了表格/列表/代码块等结构化格式
- [ ] 信息分层清晰
- [ ] 符合精炼模板格式

---

## 评分阈值

| 评分 | 行为 |
|------|------|
| < 60 | 🛑 **拦截**！进入 Loop N+1 重新学习（最多 3 次循环） |
| 60-79 | ⚠️ **警告**，标记"待改进"，允许继续但需记录改进点 |
| ≥ 80 | ✅ **通过**，产出 Artifact |

---

## 循环门检流程

```
学习完成 → 质量评分 →
├─ < 60 → 分析失败类型 →
│   ├─ 信息不足 → 回到 STEP 1 重新搜索
│   ├─ 理解不深 → 回到 STEP 2 重新阅读
│   ├─ 提炼不好 → 回到 STEP 3 重新提炼
│   └─ 结构不好 → 回到 STEP 3 重新整理
│   进入 Loop N+1（最多 3 次）
├─ 60-79 → 标记"待改进" → 继续产出 Artifact
└─ ≥ 80 → 通过 → 产出 Artifact
```

超过 3 次循环仍未通过 → 报告用户请求协助
