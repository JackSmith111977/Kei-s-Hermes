# 学习质量评分细则 v3.0

> **v3.0 更新**：增加递进评分规则、最小循环检查、子代理仲裁阈值

---

## 评分公式

```
质量分 = 信息覆盖度(30) + 交叉验证(25) + 可操作性(25) + 结构完整度(20)
```

满分 100 分。

---

## 递进评分规则（v3.0 新增）

| 循环次数 | 等级名称 | 通过门槛 | 递进惩罚 | 含义 |
|:---:|:---|:---:|:---:|:---|
| 第 1 次 | 🌐 广度扫描 | raw ≥ 60 | 0 | 强制进入第 2 轮 |
| 第 2 次 | 🔍 深度挖掘 | effective ≥ 60（raw ≥ 70） | -10 | 必须更严格 |
| 第 3 次 | ✨ 精炼优化 | effective ≥ 60（raw ≥ 80） | -20 | 精雕细琢 |

**最小循环次数**：L2 (R1/R2/R3) 和 L3 各自至少 1 次完整循环。
**超过 3 次**：报告用户请求协助。

---

## 维度 1：信息覆盖度（0-30 分）

| 分数 | 标准 |
|------|------|
| 25-30 | 覆盖了主题的各个方面，包含核心概念、最新进展、实践案例 |
| 18-24 | 覆盖了主题的主要方面，缺少 1-2 个重要维度 |
| 10-17 | 只覆盖了主题的部分方面，缺少多个重要维度 |
| 0-9 | 信息严重不足，只找到 1-2 个来源 |

**检查项**：
- [ ] 搜索了 ≥3 个高质量来源
- [ ] 覆盖了官方文档/权威社区
- [ ] 包含了时效性信息（版本、日期）

---

## 维度 2：交叉验证得分（0-25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 核心结论有 ≥3 个独立来源佐证 |
| 15-19 | 核心结论有 2 个独立来源佐证 |
| 8-14 | 核心结论只有 1 个来源，标注"待验证" |
| 0-7 | 无交叉验证，直接使用单一来源 |

**检查项**：
- [ ] 每个核心结论至少 2 个来源确认
- [ ] 矛盾信息以官方文档为准
- [ ] 标注了不确定信息

---

## 维度 3：可操作性（0-25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 提炼了明确的步骤、配置示例、避坑指南 |
| 15-19 | 有部分操作步骤，但缺少示例或避坑 |
| 8-14 | 只有概念描述，没有操作步骤 |
| 0-7 | 只有碎片信息，无法指导实践 |

**检查项**：
- [ ] 有具体的操作步骤或配置示例
- [ ] 有常见问题或避坑指南
- [ ] 有验证方法

---

## 维度 4：结构完整度（0-20 分）

| 分数 | 标准 |
|------|------|
| 17-20 | 结构完整：核心概念+步骤+避坑+示例+来源 |
| 12-16 | 结构较完整，缺少 1-2 个部分 |
| 6-11 | 只有基本结构，缺少多个重要部分 |
| 0-5 | 无结构，只是信息堆砌 |

**检查项**：
- [ ] 有清晰的标题和层次结构
- [ ] 包含了核心概念
- [ ] 包含了操作步骤
- [ ] 包含了来源和参考

---

## 子代理仲裁阈值（v3.0 新增）

| 评分区间 | 行为 |
|:---:|:---|
| ≥ 80 | ✅ 直接通过 |
| 70-79 | ⚠️ 轻微警告，标记"待改进"后通过 |
| 60-69 | ⚠️ 边界通过 + 第 1 轮强制进入第 2 轮 |
| **40-59** | **🔴 启动子代理仲裁** |
| < 40 | 🛑 强制拦截，自动回退 |

**子代理仲裁流程**（仅 40-59 区间触发）：
1. 记录门禁失败项
2. 启动子代理作为独立裁判
3. 子代理读取学习产出文件
4. 子代理判 pass → 进入下一步（即使 score < 60）
5. 子代理判 fail → 自动 regress 回退 + 记录原因
6. 将仲裁结果写入日志
