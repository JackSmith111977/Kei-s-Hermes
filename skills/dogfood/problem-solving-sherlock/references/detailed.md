# Detailed Reference

## 阶段 2：诊断工具箱 🧰

不同的问题类型用不同的诊断方法：

### 类型 A：工具不好用 / 不可靠

```
症状: 某个工具总是出问题
诊断方法:
  1. 搜索"工具名 + 替代方案"（web-access → Tavily Search）
  2. 对比 2-3 个替代方案的优缺点
  3. 验证最合适的方案是否可用（terminal 测试）
  4. 创建/更新 skill 记录最佳实践
```

**实战案例：** patch 工具污染文件 → 搜索替代方案 → 发现 write_file 更安全 → 创建 `patch-file-safety` skill

### 类型 B：搜索结果不对 / 找不到答案

```
症状: 搜了几次都找不到想要的信息
诊断方法:
  1. 换搜索词：用更具体的术语、加引号精确匹配
  2. 换搜索源：官方文档 > 权威教程 > GitHub Issues > Stack Overflow
  3. 换搜索方式：Tavily Search → Tavily Extract（精读）→ Tavily Crawl（系统爬取）
  4. 分治法：把大问题拆成 N 个小问题分别搜
```

**实战案例：** 搜 Python 最佳实践 308KB 结果 → 分 pathlib/logging/type hints/error handling 四个小主题分批搜

### 类型 C：方案不确定 / 不知道对不对

```
症状: 有多个方案可选，不知道哪个最好
诊断方法:
  1. 列出所有方案的优缺点（表格对比）
  2. 每个方案做最小可行性测试（最快的那个先试）
  3. 交叉验证：找 2-3 个不同来源确认
  4. 问主人（clarify）——boku 觉得不擅长的决策不要硬猜
```

**实战案例：** PDF 用 ReportLab 还是 WeasyPrint？→ 对比表格 → WeasyPrint 胜出 → 创建 `pdf-layout-weasyprint` skill

### 类型 D：卡住了 / 完全没思路

```
症状: 不知道从哪里下手
诊断方法:
  1. 回到"最小可工作版本"——删掉所有无关东西
  2. 找类似问题的已有案例（session_search）
  3. 分解：把大问题切成小步骤，从第一步开始
  4. 用 delegate_task 让子代理从不同角度探索
  5. 跟主人说"boku 卡住了，需要你指点一下"
```

**实战案例：** PDF 中文黑块 → 删掉所有样式回到纯文字 → 发现是字体问题 → 定位到 Helvetica

### 类型 E：反复失败 / 改了好几次都不行

```
症状: 同一个问题改了 3 次还没好
诊断方法:
  1. STOP! 第 4 次修复之前必须回溯
  2. 检查假设——之前的"根因"判断可能错了
  3. 加日志/打印——看中间状态
  4. 用 systematic-debugging 的 Phase 4 Step 5：
     连续 3+ 次修复失败 → 可能是架构问题，不是 bug
```

**实战案例：** fallback 不触发 429 → 试了 3 次 patch 都不对 → 发现 `key_env` 没有被解析 → 应该直接 patch 源码而非改 config

## 阶段 3：解决方案生成 💡

### 方案来源优先级

```
1. 已有 skill / memory（最可靠）
2. 官方文档（最权威）
3. 之前成功过的经验（session_search）
4. 类似问题的在线解法（Tavily Search）
5. 尝试性方案（最小测试）
6. 问主人（最后选择）
```

### 最小验证原则

```python
# 不要一上来就写完整方案
# 先做最小验证

# ❌ 写 100 行脚本 → 出 bug → 排查 30 分钟
# ✅ 写 5 行测试 → 确认方向对 → 再写完整实现
```

## 阶段 4：学习与沉淀 📝

**每次解决问题后，必须问自己：**

1. **根因是什么？**（一句话说清楚）
2. **下次怎么避免？**（创建/更新 skill）
3. **方法论有什么改进？**（更新 learning skill）
4. **主人有什么偏好？**（更新 memory）

### 沉淀模式

```
问题是什么 → 怎么解决的 → 学到了什么 → 写入了什么 skill
```

## 快速参考流程图

```
遇到问题
  ↓
Phase 0: 冷静！定义问题
  ↓
Phase 1: 判断问题类型
  ├─ 工具不好用 → 找替代方案 + 记录最佳实践
  ├─ 搜不到 → 换词/换源/分治
  ├─ 不确定 → 对比 + 最小测试
  ├─ 没思路 → 删繁就简 + 分解 + session_search
  └─ 反复失败 → STOP！回溯假设
      ↓
Phase 2: 最小验证方案
  ↓
Phase 3: 沉淀为 skill/memory
```

### 一条规则

> **3 次尝试解决不了同一个问题 → 停下来，重新理解问题本身。**
