## 执行流程

### 前置确认（delete-safety B 级）
1. 列出所有目录 + 确认每个目录的大小/内容/修改时间 → 汇报给 owner
2. Owner 回复「都删除」→ 走备份流程

### 备份（踩坑记录）
- 用 `cp -r` 逐个备份小目录到 `~/.hermes/disk-cleanup/` ✅
- ⚠️ **docling-env (5.7GB, 45065 文件)** → `cp -r` 超时！整个脚本中断
- 教训：大于 5GB 的目录不应 cp 备份，应当直接用 `tar` 或跳过备份

### 执行删除
分两轮：
1. 第一轮（含备份的脚本）→ 因 docling-env 超时，删除语句未被执行
2. 第二轮（单独 `rm -rf`）→ 全部成功

### 验证
- 逐个 `ls -d` 检查 → 确认不存在
- 最终磁盘：32GB/50GB (68%)，释放约 4GB

## 关键教训
1. 备份大目录（>5GB）→ 用 `tar` 或跳过备份，不要用 `cp -r`
2. 先备份再删除的脚本结构 → 备份和删除应该分离，避免一个失败影响另一个
3. Owner 的确认模式 → 先列清单 → 等批准 → 备份 → 执行 → 验证
