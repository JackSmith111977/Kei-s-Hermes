# 信息清洗方法

> 本文件记录 learning skill 积累的信息清洗和去噪技巧，每次学习后更新。

## 清洗步骤

### 1. 去除噪声
- 删除营销内容、广告、推广语
- 删除重复信息（多个来源重复的内容保留一份）
- 删除与主题无关的延伸内容

### 2. 提取关键信息
- 核心概念定义
- 代码示例和配置片段
- API 端点、参数、返回值
- 最佳实践和反模式
- 常见错误和解决方案

### 3. 交叉验证
- 矛盾信息：用官方文档或最新来源验证
- 过时信息：检查版本号、API 变更日志
- 不完整信息：补充缺失的关键点

### 4. 结构化呈现
- 用表格对比多个方案
- 用列表呈现步骤和要点
- 用代码块展示示例
- 用引用块标注注意事项

## 常见噪声类型及处理方法

| 噪声类型 | 特征 | 处理方法 |
|----------|------|----------|
| 营销内容 | "最佳"、"领先"、"革命性" | 直接删除，只保留技术细节 |
| 冗长背景 | 大量历史介绍、公司背景 | 精简为 1-2 句话 |
| 重复示例 | 多个相似代码示例 | 保留最完整/最简洁的一个 |
| 过时 API | 已废弃的函数/端点 | 标注"已废弃"，提供新 API |
| AI 生成痕迹 | 模板化语言、缺乏细节 | 交叉验证，补充具体信息 |
