## 一、9 种 Skill 类型

| 类型 | 适用场景 | 核心特征 |
|------|---------|---------|
| **库/API** | 调用外部库、REST API、CLI 工具 | 参数说明、错误处理、认证方式 |
| **产品验证** | 测试产品功能、验收标准 | 测试用例、验收清单、边界条件 |
| **数据分析** | 数据清洗、统计分析、可视化 | 数据源、处理流程、输出格式 |
| **代码生成** | 生成代码、脚手架、模板 | 语言/框架、代码规范、示例 |
| **文档生成** | 生成/排版各类文档 | 格式规范、设计原则、模板 |
| **代码审查** | 审查代码质量、安全 | 审查清单、评分标准、修复建议 |
| **部署** | CI/CD、环境配置、发布 | 部署流程、回滚策略、监控 |
| **团队规范** | 编码规范、流程规范 | 规则列表、检查工具、例外处理 |
| **调研整理** | 信息收集、竞品分析 | 信息来源、整理模板、评估标准 |

## 二、5 种设计模式

### 1. 📦 Tool Wrapper（工具包装）
- **理念**：按需注入 Token，只加载需要的部分
- **Token 效率**：60-80%（vs 全量加载）
- **适用**：库/API 类 Skill，文档很长但每次只用一小部分
- **实现**：将文档拆分为多个小文件，按需加载

### 2. 🏭 Generator（生成器）
- **理念**：模板驱动，结构预定义
- **确定性**：95%（结构固定，只需填充内容）
- **适用**：有固定格式的文档/代码/报告
- **实现**：创建 templates/ 目录，预定义模板

### 3. 🔍 Reviewer（审查器）
- **理念**：审查分离，先评分再提建议
- **流程**：加载 → 评估 → 打分 → 建议
- **适用**：代码审查、文档审查、质量评估
- **实现**：定义评估维度 + Grader 等级

### 4. 🎤 Inversion（反转）
- **理念**：先采访用户，再开始工作
- **效果**：返工率从 75% → 25%
- **适用**：需求不明确、需要用户决策
- **实现**：先提问澄清，确认后再执行

### 5. 🔄 Pipeline（流水线）
- **理念**：强制顺序执行，防止跳步
- **效果**：遗漏率从 90% → 5%
- **适用**：多步骤复杂任务
- **实现**：定义阶段 + 检查点 + 硬性门控

## 三、9 阶段工作流

```
Stage 1: 需求分析
  → 明确任务目标、成功标准、约束条件

Stage 2: 内容结构
  → 确定大纲、模块划分、依赖关系

Stage 3: 设计规范
  → 确定风格、格式、技术选型

Stage 4: 内容填充
  → 填充具体内容、代码、数据

Stage 5: 视觉润色
  → 对齐、间距、装饰、优化

Stage 6: 质量验证
  → 对照清单逐项检查

Stage 7: 输出交付
  → 生成文件、发送、存档

Stage 8: 文档更新
  → 更新 Skill 文档、记录经验

Stage 9: 评估迭代
  → 打分、记录历史、持续改进
```

---

## 四、Stage 9 评估体系

### 4.1 evals.json
- 定义评估维度（E1-E6）
- 每项权重和检查点
- 打分标准（0-1）

### 4.2 Grader
- PASS（≥0.90）：直接交付
- WARN（0.75-0.89）：小修后交付
- FAIL（0.60-0.74）：返工
- CRITICAL（<0.60）：重新设计

### 4.3 history.json
- 版本追踪
- 每次修改记录
- 分数变化趋势

### 4.4 benchmark.json
- 盲对比测试
- 多方案横向比较
- 最优方案选择

## 五、关键设计原则

### 5.1 主文件 < 500 行
- SKILL.md 保持精简
- 详细内容放 references/
- 按需加载，避免 Token 浪费

### 5.2 模式组合
- 复杂 Skill 可以组合多种模式
- 例如：Pipeline + Generator + Reviewer
- 模式之间用决策树连接

### 5.3 硬性门控
- Pipeline 模式必须有门控
- 前一阶段未完成不得进入下一阶段
- 返工必须回到对应阶段

### 5.4 经验积累
- 每次任务完成后记录经验
- 站点模式、模板、常见错误
- 形成可复用的知识库

## 六、Skill 创建 Checklist

创建新 Skill 时对照检查：

- [ ] 确定 Skill 类型（9 种之一）
- [ ] 选择设计模式（1-2 种）
- [ ] 编写 SKILL.md（< 500 行）
- [ ] 创建 references/ 详细文档
- [ ] 创建 templates/ 模板（如 Generator 模式）
- [ ] 定义 evals.json 评估维度
- [ ] 定义 pipeline-progress.yaml（如 Pipeline 模式）
- [ ] 编写 Red Flags（常见错误）
- [ ] 编写验证清单
- [ ] 测试 Skill 可用性
- [ ] 记录初始版本到 history.json

```
├── references/           # 详细参考文档
│   ├── cdp-api.md
│   ├── evals.json        # 评估维度

│   └── site-patterns/    # 站点经验

├── templates/            # 模板（Generator 模式）
│   └── default.md
├── scripts/              # 脚本
│   └── check-deps.mjs
└── assets/               # 静态资源
```
