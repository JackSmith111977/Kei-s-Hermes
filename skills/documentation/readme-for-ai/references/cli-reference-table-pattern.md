# CLI Reference Table Pattern — 三列格式 + 编号章节

> 使 README 中的 CLI 命令参考对 AI Agent 直接可解析。

## 核心设计

每个命令的文档条目使用 **三列 Markdown 表格**：

```markdown
| 命令 | 作用 | 关键参数 |
|:-----|:-----|:---------|
| `install <dir>` | 从目录安装能力包 | `--dry-run`, `--target hermes\|opencode` |
| `remove <name>` | 卸载已安装的包 | `--target` |
```

### 为什么是三列？

| 列 | AI 用途 | 内容约束 |
|:---|:--------|:---------|
| **命令** | 提取命令模板 | 包含变量占位符 `<arg>` |
| **作用** | 语义匹配 | ≤10 字，动词开头 |
| **关键参数** | 参数补全 | 逗号分隔，可含 `\|` 替代方案 |

## 章节编号规则

使用中文数字前缀（而非序号数字）让 AI 区分章节编号与列表编号：

```
✅ 一、项目身份
✅ 二、快速安装
✅ 三、能力包列表
❌ 1. 项目身份     ← 与列表编号冲突
❌ 项目身份        ← AI 难以建立索引
```

## 完整速查表的结尾标记

速查表末尾需要一个可正则匹配的边界信号，帮助 AI 判断「表格结束」：

```markdown
## 十一、相关项目
| 项目 | 关系 |
|:-----|:------|
| [SRA] | CAP Pack 的运行时推荐搭档 |
```

AI 通过扫描 `^## ` 标题检测表格边界。

## 实战示例

见 `hermes-cap-pack/README.md`（2026-05-14 版本）：
- 编号章节：一~十一
- 三列 CLI 速查表：第 10 节「完整命令速查表」
- 每节末尾的隐式验证命令：第 4 节末尾的 cap-pack status 示例
