## 一、UI 设计风格全景

### 1.1 主流风格速览

| 风格 | 视觉特征 | 可用性 | 代表产品 | 最适合 |
|:----|:---------|:-----:|:--------|:------|
| **扁平化设计** | 二维、无阴影/渐变、明亮色彩、清晰排版 | ✅ 高 | iOS 7+、Windows 8 | 几乎所有产品 |
| **Material Design** | 纸张隐喻、阴影层级、动态配色、物理动效 | ✅ 高 | Google 全家桶、Android | 企业级/跨平台 |
| **玻璃拟态** | 磨砂玻璃、backdrop-filter:blur、半透明 | ⚠️ 中 | macOS Big Sur、Win11 | 卡片/导航/桌面系统 |
| **新拟物化** | 同色系、左上亮右下暗投影、柔和立体 | ❌ 低 | Dribbble 概念 | 仅局部点缀 |
| **粗野主义** | 黑粗边框、硬阴影、高饱和撞色、系统字体 | ⚠️ 中 | Linear、独立站点 | 品牌差异化/创意 |
| **极简主义** | 大量留白、黑白灰+点缀色、无衬线字体 | ✅ 高 | Apple、MUJI、Notion | 内容型/高端品牌 |
| **侘寂美学** | 天然材质、大地色系、不对称、粗糙感 | ✅ 高 | 手工品牌、禅意风格 | 情感化/慢设计 |

### 1.2 风格演变简史

```
2000s 拟物化 → 2013 扁平化(iOS 7) → 2014 Material Design 
→ 2016 扁平2.0(微阴影+渐变) → 2019 新拟物化 → 
2020 玻璃拟态(macOS Big Sur) → 2021 Material You(M3) → 
2025 M3 Expressive(情感化+大胆色彩)
```

### 1.3 风格选择决策树

```
项目类型？
├─ 企业/工具型 → Material Design / 扁平化
├─ 内容展示型 → 极简主义
├─ 品牌差异化 → 粗野主义 / 玻璃拟态(局部)
├─ 移动端 → 扁平化 + Material 组件规范
└─ 创意型 → 小众风格(侘寂/Swiss)混搭
```

## 二、UX 交互定律速查

### 2.1 七大定律

| 定律 | 公式 | UX 操作 |
|:----|:-----|:--------|
| **菲茨定律** | T = a + b log₂(D/W+1) | 重要按钮≥44px，放屏幕边角 |
| **希克定律** | RT = a + b log₂(n) | 导航项≤5个，分步操作 |
| **米勒定律 7±2** | 短时记忆7±2组块 | 信息分组展示 |
| **格式塔原则** | 整体≠部分之和 | 接近=相关，相似=同类 |
| **雅各布定律** | 用户依赖已有经验 | 遵循平台惯例，不重新发明 |
| **泰思勒定律** | 复杂不可消除只可转移 | 复杂度留给开发，简单给用户 |
| **莱斯托夫效应** | 特殊易被记住 | CTA 按钮用特殊颜色 |

### 2.2 无障碍基线 (WCAG 2.2 AA)

| 优先级 | 检查项 | 标准 |
|:-----:|:-------|:----|
| P0 | 文本对比度 | ≥ 4.5:1 |
| P0 | 键盘可操作 | 所有功能 Tab 可达 |
| P1 | 焦点状态 | 可见的 focus ring |
| P1 | 非文本替代 | 图片有 alt text |
| P2 | 避免闪烁 | 无 3 次/秒闪烁 |

## 三、风格陷阱速查

| 陷阱 | 症状 | 修复 |
|:----|:-----|:-----|
| 纯扁平过度 | 按钮不可见 | 加微阴影(扁平2.0) |
| 玻璃滥用 | 页面可读性差 | 仅用于卡片/导航，不用于按钮 |
| 新拟态全站 | 所有元素都像可点击 | 仅局部点缀，别铺开 |
| 粗野主义硬怼 | 用户找不到操作入口 | "看起来硬，用起来正常" |
| 忽视无障碍 | 色盲用户无法使用 | 用颜色+图标双重编码 |

## 四、情感化设计三层模型 (Don Norman)

```
本层(Visceral) → 视觉第一印象（色彩/排版/动效）
行为层(Behavioral) → 交互愉悦度（反馈/过渡/响应）
反思层(Reflective) → 使用后记忆（成就/身份/故事）
```

**实践检查**：一个设计好不好，三层都要覆盖。只有视觉没有交互=花瓶；只有功能没有情感=机器。
